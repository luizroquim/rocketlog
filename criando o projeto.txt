*CRIANDO O PROJETO*

1- npm init - y (criar  o package json)
2- npm i express@4.19.2 (instalar Express)(package-lock.json)
3- criar arquivo server.ts
4- criar arquivo app.ts
5- npm i --save-dev @types/express@4.17.21 (instalar tipagem para usar TypeScript no express)
6- npm i typescript@5.5.4 @types/node@20.14.12 -D ( instalar typescript e tipagem para o node)
7- npm i tsx@4.16.2 -D é uma ferramenta poderosa e moderna que permite executar arquivos TypeScript (.ts) diretamente no Node.js, sem precisar compilá-los manualmente para JavaScript antes.
8- npx tsc --init (Ele gera automaticamente um arquivo chamado tsconfig.json)
9- configurar script em package para executar a aplicação.
10 - Criar pasta middlewares  para tratar as exceções da aplicação
11- Criar arquivo error-handling.ts 
12- Criar arquivo AppError.ts (criar erros personalizados e padronizados na sua aplicação.)
13- npm i express-async-errors@3.1.1 (Ele permite que o Express capture erros que acontecem dentro de funções async (assíncronas) e os envie automaticamente para o seu middleware de erro (aquele errorHandling que você criou na pergunta anterior).)
14- importar error Handling no app.ts
15-  npm i zod@3.23.8 ( instalar zod le serve para garantir que os dados que o seu usuário envia (pelo formulário, JSON, etc.) estão no formato exato que você espera, antes mesmo de você tentar usar esses dados no banco de dados.) 
16 - Criar rota e controller de Usuário
17- configura controller
18- configura rota 
19- criar index para centralizar as rotas
20- importar as rotas para dentro do arquivo app.ts

*BANCO DE DADOS*
Organizando o docker para rodar o banco de dados em um container
1- docker ps -a(listar os contaieners)
2- docker rm -f + id (apagar container)
3- docker rmi +id (apagar imagem)
4- docker image ls (lista imagens)
5- Criar pastar docker-compose.yml
6- docker-compose up - d ( executa o dockercompose)
7- testar o banco de dados  no beekeeper

Instalando e configurando o Prisma

1-npm install prisma --save-dev 
2-npm install @prisma/client --save-dev (Client para que sua aplicação possa se conectar ao banco de dados e executar operações (queries, inserções, atualizações) de forma segura e tipada.)
3-npx prisma init --datasource-provider postgresql (Cria o diretório 'prisma/' para armazenar o esquema.Cria o arquivo 'prisma/schema.prisma' com a configuração do DB.Cria o arquivo '.env' para armazenar a string de conexão (DATABASE_URL).)
4-Configurar arquivo env  (variavel de ambiente)
5- "tsx watch --env-file .env src/server.ts" Adicionar variavel de ambiente no script de inicilização  
6-Colocar .env dentro do gitignore
